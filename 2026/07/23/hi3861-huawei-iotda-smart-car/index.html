基于 Hi3861 与华为云 IoTDA 的智能小车云端协同系统

2026-07-23 Hi3861 HarmonyOS 华为云 IoTDA MQTT MySQL

本项目围绕 Hi3861 智能小车搭建了一套端、云、服务与多终端协同的物联网系统。小车能够完成手动方向控制、循迹、避障、环境数据采集和蜂鸣器告警，并通过 MQTT 接入华为云 IoTDA；手机端、平板端和网页管理控制台可查看实时数据、下发控制命令，后端同时将设备数据与命令记录保存到 MySQL。

一、项目目标

传统智能小车通常只完成本地传感器控制，本项目进一步打通完整的云端链路：

  • Hi3861 负责传感器采集、电机驱动和本地控制策略。
  • 华为云 IoTDA 负责设备接入、属性上报、设备影子和命令下发。
  • 后端服务负责封装云端接口、同步设备数据和保存命令日志。
  • HarmonyOS 手机端、平板端和网页管理控制台负责数据展示与远程控制。
  • MySQL 持久化保存设备读数、运行状态和命令执行记录。

二、硬件端功能

小车固件基于 OpenHarmony 轻量系统开发，将传感器采集、云端通信和车辆控制拆分为不同任务，避免网络通信阻塞电机控制。

1. 三种工作模式

  • 手动模式：接收前进、后退、左转、右转和停止命令。
  • 循迹模式：根据红外传感器状态自动修正运动方向。
  • 避障模式：根据障碍物距离执行停车或转向策略。

2. 环境监测与告警

设备周期性采集温度、湿度和距离数据。当温度超出设定阈值时触发告警状态，并可联动蜂鸣器。阈值、工作模式、运动方向和告警状态均通过统一属性模型在设备端与云端之间同步。

3. MQTT 通信

Hi3861 连接 Wi-Fi 后接入华为云 IoTDA，完成以下通信流程：

  • 周期上报 temperature、humidity、distance、direction 和 work_mode。
  • 订阅云端命令主题，解析控制命令并投递到本地消息队列。
  • 执行电机或蜂鸣器动作后返回命令响应，便于云端判断执行结果。

三、HarmonyOS 多终端应用

手机端使用 DevEco Studio 开发，首页直接展示设备在线状态、当前模式、温湿度、报警状态和雷达距离。底部导航提供实时数据、历史记录、设备控制、报警中心和个人中心等页面。

设备控制页面可以切换手动、循迹和避障模式，并下发方向、停车及蜂鸣器控制命令。报警中心支持配置温度规则和蜂鸣器屏蔽，适合在模拟器中演示完整交互流程。

平板端采用更适合大屏的双栏布局，在同一页面显示设备概览、趋势数据和控制面板，实现同一套业务在不同终端上的自适应展示。

四、网页管理控制台

网页端面向设备管理员，重点提高数据查看和重复操作效率，主要包括：

  • 实时温湿度、距离、运行模式和设备在线状态。
  • 最近数据趋势与数据库记录数量。
  • 手动方向控制、模式切换、紧急停车和蜂鸣器控制。
  • 自定义云端命令及参数 JSON 下发。
  • 命令时间、参数、云端命令 ID 和执行状态记录。

五、云数据转存与数据库设计

后端定时读取 IoTDA 设备影子，将最新属性转换为统一的数据对象后写入 MySQL。前端不直接访问云端密钥，而是通过后端 REST API 获取设备数据和下发命令，从而将云端认证信息与用户界面隔离。

核心数据分为两类：

数据表 主要内容
cloud_reading 设备 ID、服务 ID、温湿度、方向、上下限、工作模式、采集时间和原始属性
command_log 命令名称、请求参数、下发时间、云端命令 ID、执行状态和响应信息

数据库记录保留云端采集时间与本地入库时间，方便检查数据同步延迟和追踪异常记录。

六、关键数据链路

以手机端点击“前进”为例，完整流程如下：

  1. HarmonyOS 应用向后端提交方向控制请求。
  2. 后端调用华为云 IoTDA 命令下发接口，并写入命令日志。
  3. IoTDA 通过 MQTT 将命令推送至 Hi3861。
  4. 固件解析命令并控制电机前进。
  5. 设备重新上报 direction 和运行状态。
  6. 后端同步设备影子并写入 MySQL，手机端与网页端刷新显示结果。

这条链路将“按钮点击”与“设备真实状态”区分开来，界面展示以设备上报结果为准，而不是仅在本地修改文本。

七、项目实现结果

  • 完成 Hi3861 固件编译与烧录，设备能够连接 Wi-Fi 和华为云 IoTDA。
  • MQTT 属性上报与命令订阅成功，实测温度约为 24～26 ℃、湿度约为 66%～73%。
  • 手机端能够查看实时数据并控制小车前进、后退、转向和停止。
  • 实现手动、循迹和避障三种运行模式，并加入温度阈值与蜂鸣器告警逻辑。
  • 后端完成设备影子同步、历史数据查询和命令记录，MySQL 可持续保存采集结果。
  • 完成手机端、平板端和网页管理控制台三种界面，形成统一的数据展示与控制体验。

八、项目总结

这个项目不只是完成小车运动控制，而是把嵌入式开发、MQTT 通信、华为云设备模型、后端接口、数据库和 HarmonyOS 多终端应用连接成一套完整系统。开发过程中重点解决了设备属性命名一致性、命令响应、串口烧录占用、设备影子同步和数据库时间字段等问题。

后续可以继续加入轨迹可视化、离线告警、自动巡航策略和基于历史数据的环境趋势分析，使系统从课程项目进一步扩展为可持续维护的物联网应用。
